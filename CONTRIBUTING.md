## דרכי תרומה לפרויקט

תודה שבחרתם לתרום! מטרת הפרויקט היא לספק בסיס קל להפעלה ותחזוקה של בלוג Jekyll תחת חשבון GitHub אישי. כדי לשמור על תהליך נקי וידידותי, אנא פעלו לפי ההנחיות הבאות.

### לפני שמתחילים

- בדקו את `ISSUES` הפתוחים כדי לוודא שהתרומה שלכם אינה כפולה.
- אם מדובר ברעיון חדש, פתחו דיון קצר או פתחו Issue new feature והציגו את הערכתכם.
- ודאו שהסביבה המקומית שלכם מוגדרת לפי ההנחיות בקובץ `JEKYLL_GUIDE_HE.md`.

### זרימת עבודה מומלצת

1. צרו Fork למאגר (אם אינכם חברי צוות).
2. צרו Branch חדש ושמותיו באופן תיאורי, לדוגמה: `feature/add-search`.
3. בצעו את השינויים (כולל בדיקות רלוונטיות).
4. הריצו `bundle exec jekyll build` לוודא שאין שגיאות.
5. עדכנו את התיעוד (`README.md`, `VERSIONS.md`, `CHANGELOG.md`) במידת הצורך.
6. שלחו Pull Request ותארו את המניעים והשינויים שביצעתם.

### הנחיות לקוד ולתיעוד

- שמרו על עקביות בסגנון (עיינו בקבצים הקיימים).
- הוסיפו בדיקות או תיאורי בדיקות עבור שינויים משמעותיים.
- שמרו על שפה עברית במסמכים, אלא אם מדובר בציטוטים או רישיונות רשמיים.
- עדכנו את `CHANGELOG.md` עם סעיף "Unreleased" או גרסה חדשה, לפי ההנחיות שבקובץ.

### בדיקות

- השתמשו בפקודות הבאות לפני שליחת Pull Request:
  - `bundle exec jekyll build`
  - `bundle exec jekyll doctor`
- אם הוספתם תלויות חדשות, וודאו שכוללתן מתועדת בקובץ `Gemfile` והסבירו מדוע.

### סקירת קוד

- אנו מצפים לתגובות ענייניות, עם דגש על שיפור איכות, נגישות ויציבות.
- זמני תגובה לסקירות: עד 5 ימי עבודה. אם עבר הזמן, הזכירו את המתחזקים בעדינות.

### קבלות לעדכונים

לאחר שה-PR מאושר ומוזג:

- עדכנו את `VERSIONS.md` בגרסה החדשה.
- ודאו שהשחרור תועד ב-`CHANGELOG.md`.
- במידת הצורך, פרסמו פוסט בבלוג על השינוי הגדול.
